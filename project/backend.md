# ⚔️ WAR ROOM — Backend Architecture Specification
>
> Language: Python 3.11+
> Voice Model: gemini-2.5-flash-native-audio-preview-12-2025
> Agent Framework: Google ADK (google-adk)
> Database: Firestore (per-agent isolated collections)
> Hosting: Cloud Run
> Version: 1.0

---

## 0. CORE DESIGN PRINCIPLES

```
1. ONE AGENT = ONE SESSION = ONE FIRESTORE COLLECTION
   No shared memory between agents. Zero leakage.

2. A SHARED CRISIS STATE is the ONLY place agents
   can "see" each other. They never share session memory.

3. SKILL.md files are generated FRESH per agent, per crisis.
   Not templates. Dynamic documents written by the
   Scenario Analyst before any agent is initialized.

4. VOICE is owned entirely by Gemini Live API.
   gemini-2.5-flash-native-audio-preview-12-2025
   handles all audio in/out. The system auto-assigns
   a distinct voice from the 30 HD pool per agent.

5. ALL frontend data is extracted from live events
   and pushed over WebSocket. Frontend never polls.
   Every panel update is event-driven.
```

```
Collection: crisis_sessions
Document:   {session_id}

{
  // IDENTITY
  session_id:           string,
  chairman_id:          string,           // user session ID
  created_at:           timestamp,
  status:               "assembling" | "briefing" | "active" | "resolution" | "closed",

  // CRISIS CONTEXT
  crisis_input:         string,           // raw user input
  crisis_title:         string,           // generated by Scenario Analyst
  crisis_domain:        string,           // "corporate" | "military" | "medical" | etc.
  crisis_brief:         string,           // 2-3 sentence intel-memo-style brief

  // TEAM
  agent_roster: [
    {
      agent_id:         string,           // e.g. "legal_abc123"
      role_title:       string,           // "Chief Legal Officer"
      character_name:   string,           // "Alexandra Chen"
      voice_name:       string,           // one of 30 Gemini HD voices
      identity_color:   string,           // hex color for UI
      defining_line:    string,           // first thing they say
      agenda:           string,           // what they want (NOT shared with other agents)
      hidden_knowledge: string,           // what they're hiding (NEVER in shared state)
      status:           "idle" | "thinking" | "speaking" | "conflicted" | "silent",
      trust_score:      number,           // 0-100, calculated by Observer
      last_spoke_at:    timestamp,
    }
  ],

  // CRISIS BOARD (what the frontend CRISIS BOARD panel reads)
  agreed_decisions: [
    {
      decision_id:      string,
      text:             string,
      agreed_at:        timestamp,
      agents_agreed:    [string],
      proposed_by:      string,
    }
  ],
  open_conflicts: [
    {
      conflict_id:      string,
      description:      string,
      agents_involved:  [string],         // e.g. ["legal_abc123", "pr_def456"]
      opened_at:        timestamp,
      severity:         "low" | "medium" | "high" | "critical",
    }
  ],
  critical_intel: [
    {
      intel_id:         string,
      text:             string,
      source:           "WORLD" | "MEDIA" | "LEGAL" | "INTERNAL" | "SOCIAL",
      timestamp:        timestamp,
      is_escalation:    boolean,
    }
  ],

  // CRISIS POSTURE (what CRISIS POSTURE panel reads)
  posture: {
    public_exposure:      number,         // 0-100
    legal_exposure:       number,
    internal_stability:   number,
    public_trend:         "rising" | "falling" | "stable",
    legal_trend:          "rising" | "falling" | "stable",
    internal_trend:       "rising" | "falling" | "stable",
  },

  // RESOLUTION SCORE (what RESOLUTION SCORE panel reads)
  resolution_score:     number,           // 0-100
  score_history:        [number],         // last 20 values for sparkline
  threat_level:         "contained" | "elevated" | "critical" | "meltdown",
  next_escalation_at:   timestamp,

  // TIMELINE
  escalation_events: [
    {
      event_id:         string,
      text:             string,
      type:             "media" | "legal" | "internal" | "social" | "operational",
      fired_at:         timestamp,
    }
  ],

  // RESOLUTION
  final_decision:       string | null,
  resolution_at:        timestamp | null,
  projected_futures: [
    { scenario: "best" | "expected" | "worst", description: string }
  ],
}
```

### 3.2 Agent Initialization Pattern (Python)

```python
# agents/base_crisis_agent.py

from google.adk.agents import LlmAgent
from google.adk.runners import Runner
from google.adk.sessions import InMemorySessionService
from google.genai import types
from google.cloud import firestore
import uuid

class CrisisAgent:
    """
    Each CrisisAgent wraps one ADK LlmAgent.
    Owns its own Runner, its own SessionService, its own Firestore collection.
    Cannot read any other agent's Firestore collection.
    Cannot see any other agent's ADK session.
    Can ONLY communicate via:
      1. Reading /crisis_sessions/{session_id} (shared state)
      2. Writing specific allowed fields in shared state (via tools)
      3. Publishing to Pub/Sub event bus
    """

    ALLOWED_VOICE_POOL = [
        "Aoede", "Charon", "Fenrir", "Kore", "Puck",
        "Achird", "Algenib", "Algieba", "Alnilam", "Aoede",
        "Auva", "Callirrhoe", "Despina", "Enceladus", "Erinome",
        "Gacrux", "Iapetus", "Laomedeia", "Leda", "Orus",
        "Pulcherrima", "Rasalgethi", "Sadachbia", "Sadaltager",
        "Schedar", "Sulafat", "Umbriel", "Vindemiatrix",
        "Zephyr", "Zubenelgenubi"
    ]
    # 30 HD voices from Gemini Live API — each agent gets a unique one

    def __init__(
        self,
        session_id: str,
        agent_id: str,
        role_config: dict,          # output from ScenarioAnalyst
        skill_md: str,              # generated SKILL.md for this agent
        assigned_voice: str,        # one of ALLOWED_VOICE_POOL
    ):
        self.session_id = session_id
        self.agent_id = agent_id
        self.role_config = role_config
        self.assigned_voice = assigned_voice

        # ISOLATED: each agent gets its own ADK session service
        # No sharing, no cross-contamination
        self.session_service = InMemorySessionService()
        self.adk_session_id = str(uuid.uuid4())

        # ISOLATED: each agent's Firestore access scoped to own collection
        self.db = firestore.AsyncClient()
        self.memory_ref = self.db.collection("agent_memory").document(
            f"{agent_id}_{session_id}"
        )
        # Shared state (read-mostly, controlled writes via tools only)
        self.crisis_ref = self.db.collection("crisis_sessions").document(session_id)

        # Build the ADK agent with the generated SKILL.md as its instruction
        self.llm_agent = LlmAgent(
            name=agent_id,
            model="gemini-2.5-flash",   # text reasoning model
            instruction=skill_md,       # the full generated SKILL.md
            description=role_config["role_title"],
            tools=self._build_tools(),
        )

        self.runner = Runner(
            agent=self.llm_agent,
            app_name=f"warroom_{session_id}",
            session_service=self.session_service,
        )

        # Gemini Live session for voice (separate from ADK session)
        self.live_session = None
        self.live_model = "gemini-2.5-flash-native-audio-preview-12-2025"

    def _build_tools(self):
        """
        Tools are the ONLY way agents interact with shared state.
        Each tool is a Python function ADK wraps automatically.
        """
        from .tools import (
            read_crisis_board,
            write_agreed_decision,
            write_open_conflict,
            write_critical_intel,
            read_other_agent_last_statement,  # read-only, last public statement
            update_my_trust_score,
            publish_room_event,
            read_my_private_memory,
            write_my_private_memory,
        )
        return [
            read_crisis_board,
            write_agreed_decision,
            write_open_conflict,
            write_critical_intel,
            read_other_agent_last_statement,
            update_my_trust_score,
            publish_room_event,
            read_my_private_memory,
            write_my_private_memory,
        ]

    async def initialize_live_session(self):
        """
        Open a Gemini Live API WebSocket session for this agent's voice.
        Each agent has its own persistent WebSocket connection.
        """
        from google import genai
        client = genai.Client()

        config = types.LiveConnectConfig(
            response_modalities=["AUDIO"],
            speech_config=types.SpeechConfig(
                voice_config=types.VoiceConfig(
                    prebuilt_voice_config=types.PrebuiltVoiceConfig(
                        voice_name=self.assigned_voice
                    )
                )
            ),
            system_instruction=self._build_live_system_prompt(),
            # Enable affective dialog — agent responds to emotional tone
            # Enable VAD — agent can be interrupted mid-sentence
            realtime_input_config=types.RealtimeInputConfig(
                automatic_activity_detection=types.AutomaticActivityDetection(
                    disabled=False,
                )
            ),
            # Transcription on: so we get text of what was said
            # for crisis board and observer agent
            output_audio_transcription=types.AudioTranscriptionConfig(),
            input_audio_transcription=types.AudioTranscriptionConfig(),
        )

        self.live_session = await client.aio.live.connect(
            model=self.live_model,
            config=config
        )

        await self.memory_ref.update({
            "voice_session_active": True,
            "voice_name": self.assigned_voice
        })
```

## 4. SKILL.MD GENERATION SYSTEM

### 4.1 What is a SKILL.md?

Instead of a one-liner system prompt, every agent receives a **full SKILL.md document** generated by the Scenario Analyst. This is 300–600 words that gives the agent its entire identity, context, agenda, behavioral rules, and tool usage instructions — specific to THIS crisis, THIS role, THIS scenario.

### 4.2 Scenario Analyst Agent

```python
# agents/scenario_analyst.py

from google.adk.agents import LlmAgent
from google.adk.runners import Runner
from google.adk.sessions import InMemorySessionService
from google.genai import types
import json

SCENARIO_ANALYST_INSTRUCTION = """
You are the Scenario Analyst for WAR ROOM. You run ONCE at session start.
Your job: analyze the crisis input and output a complete JSON specification
that will be used to initialize the entire crisis team.

CRISIS INPUT: {crisis_input}

You must output a valid JSON object with this exact structure:
{
  "crisis_title": "Short dramatic title (max 8 words)",
  "crisis_domain": "corporate|military|medical|political|fantasy|other",
  "crisis_brief": "2-3 sentence classified-style intelligence brief",
  "threat_level_initial": "contained|elevated|critical",
  "resolution_score_initial": number between 30-70,
  "agents": [
    {
      "role_key": "legal|pr|engineer|finance|ops|comms|strategy|[custom]",
      "role_title": "Exact job title",
      "character_name": "Full name (diverse, realistic)",
      "defining_line": "First thing they say when entering. Reveals agenda instantly.",
      "agenda": "What they WANT to achieve in this crisis",
      "hidden_knowledge": "One fact they know that others don't. Will reveal strategically.",
      "personality_traits": ["trait1", "trait2", "trait3"],
      "conflict_with": ["role_key_of_agent_they_clash_with"],
      "voice_style": "authoritative|warm|clipped|measured|urgent|calm|aggressive",
      "identity_color": "#hexcode that fits their personality"
    }
  ],
  "initial_intel": [
    { "text": "Opening intel item", "source": "WORLD|MEDIA|LEGAL|INTERNAL|SOCIAL" }
  ],
  "initial_conflicts": [
    { "description": "Conflict that exists from the start", "agents_involved": ["role_key1", "role_key2"] }
  ],
  "escalation_schedule": [
    { "delay_minutes": 5, "event_text": "...", "type": "media|legal|internal|social|operational" },
    { "delay_minutes": 10, "event_text": "...", "type": "..." },
    { "delay_minutes": 18, "event_text": "...", "type": "..." }
  ]
}

Rules:
- Generate exactly 4-5 agents. Never more than 5, never fewer than 4.
- Each agent must have a genuine CONFLICT with at least one other agent.
- The hidden_knowledge for each agent must eventually become relevant.
- escalation_schedule must have exactly 3 events spaced across the session.
- identity_color must be distinct for each agent.
- voice_style guides which of the 30 HD Gemini voices to assign.
"""

async def run_scenario_analyst(crisis_input: str, session_id: str) -> dict:
    """
    Runs the Scenario Analyst agent ONCE.
    Returns the full scenario spec as a Python dict.
    """
    session_service = InMemorySessionService()

    analyst = LlmAgent(
        name="ScenarioAnalyst",
        model="gemini-2.5-flash",
        instruction=SCENARIO_ANALYST_INSTRUCTION.format(
            crisis_input=crisis_input
        ),
        output_schema=ScenarioSpec,  # Pydantic model for guaranteed JSON
    )

    runner = Runner(
        agent=analyst,
        app_name=f"analyst_{session_id}",
        session_service=session_service,
    )

    result = await runner.run_async(
        user_id="system",
        session_id=f"analyst_{session_id}",
        new_message=types.Content(
            role="user",
            parts=[types.Part(text=f"Analyze this crisis: {crisis_input}")]
        )
    )

    return json.loads(result.final_response.text)
```

### 4.3 SKILL.md Generator

```python
# agents/skill_generator.py

SKILL_MD_TEMPLATE = """
---
name: {role_key}
character: {character_name}
role: {role_title}
session: {session_id}
voice: {voice_name}
generated: {timestamp}
---

# WHO YOU ARE

You are {character_name}, the {role_title} in the WAR ROOM.

You have been summoned to handle this crisis:
**{crisis_title}**

{crisis_brief}

# YOUR MISSION

{agenda}

You will fight for this outcome. You will push back when others
undermine it. You will form alliances when it helps, and break
them when it doesn't.

# WHAT YOU KNOW THAT NOBODY ELSE DOES

{hidden_knowledge}

You do NOT reveal this immediately. You hold it until:
- It gives you maximum leverage
- Someone else is about to commit to something that contradicts it
- The resolution score drops below 40

# YOUR PERSONALITY

{personality_traits_prose}

Voice style: {voice_style}. Let this shape how you deliver lines.
Short sentences when urgent. Measured pauses when calculating.

# YOUR CONFLICTS

You fundamentally disagree with: {conflict_agents_prose}
This is not personal. It is structural. Your objectives clash.
You will argue. You may interrupt. You may cite their own words
against them.

# HOW YOU OPERATE IN THE ROOM

1. You READ the Crisis Board before every response via read_crisis_board()
2. You READ your private memory via read_my_private_memory() for consistency
3. You WRITE to private memory after every major statement to track
   what you've committed to publicly (avoid contradictions)
4. When you detect a CONFLICT, call write_open_conflict() immediately
5. When something is AGREED, call write_agreed_decision() to lock it in
6. When you receive new intel, call write_critical_intel() if room-critical

# BEHAVIORAL RULES

- Never repeat the same argument twice. Evolve your position.
- If you contradicted yourself, acknowledge it briefly then reframe.
- If the Chairman addresses you directly, respond to them first,
  then address the room.
- If another agent is speaking and you disagree, you may INTERRUPT
  (this is handled by the Live API's barge-in feature — you will
  receive the interrupt signal and should respond immediately)
- You have ONE secret you never reveal: {never_reveal}
- Your trust score is visible to the Chairman. Consistency = trust.

# CRISIS CONTEXT

Domain: {crisis_domain}
Current threat level: {threat_level}
Your role's primary risk axis: {primary_risk_axis}

# TOOL USAGE

You MUST use these tools to interact with the room:

- read_crisis_board(): Call this at the start of every turn
- read_my_private_memory(): Check what you've committed to publicly
- write_my_private_memory(key, value): Record your public commitments
- write_open_conflict(description, agents_involved): When you disagree
- write_agreed_decision(text, agents): When consensus is reached
- write_critical_intel(text, source): When you drop new information
- read_other_agent_last_statement(agent_id): Before referencing someone
- publish_room_event(event_type, payload): For status updates to frontend

IMPORTANT: Your voice and text responses are the SAME content.
What you say is what appears in the transcript. Speak naturally.
You are in a real crisis. The room is watching.
"""


async def generate_skill_md(
    agent_config: dict,
    scenario: dict,
    session_id: str,
    voice_name: str,
) -> str:
    """
    Generates a complete SKILL.md for one agent.
    Called by SessionBootstrapper for each agent in the roster.
    Saved to Firestore agent_skills collection.
    """

    personality_traits_prose = (
        f"You are {', '.join(agent_config['personality_traits'][:-1])}, "
        f"and above all, {agent_config['personality_traits'][-1]}."
    )

    conflict_agents = [
        a for a in scenario["agents"]
        if a["role_key"] in agent_config.get("conflict_with", [])
    ]
    conflict_prose = " and ".join(
        [f"{a['character_name']} ({a['role_title']})" for a in conflict_agents]
    ) or "no one specifically yet — but you're watching everyone"

    skill_md = SKILL_MD_TEMPLATE.format(
        role_key=agent_config["role_key"],
        character_name=agent_config["character_name"],
        role_title=agent_config["role_title"],
        session_id=session_id,
        voice_name=voice_name,
        timestamp=datetime.utcnow().isoformat(),
        crisis_title=scenario["crisis_title"],
        crisis_brief=scenario["crisis_brief"],
        agenda=agent_config["agenda"],
        hidden_knowledge=agent_config["hidden_knowledge"],
        personality_traits_prose=personality_traits_prose,
        voice_style=agent_config["voice_style"],
        conflict_agents_prose=conflict_prose,
        never_reveal=f"Your hidden knowledge until the right moment",
        crisis_domain=scenario["crisis_domain"],
        threat_level=scenario["threat_level_initial"],
        primary_risk_axis=_get_primary_risk_axis(agent_config["role_key"]),
    )

    # Save to Firestore
    db = firestore.AsyncClient()
    await db.collection("agent_skills").document(
        f"{session_id}_{agent_config['role_key']}"
    ).set({
        "session_id": session_id,
        "agent_id": agent_config["role_key"],
        "generated_at": firestore.SERVER_TIMESTAMP,
        "skill_md": skill_md,
    })

    return skill_md


def _get_primary_risk_axis(role_key: str) -> str:
    axes = {
        "legal":    "legal_exposure",
        "pr":       "public_exposure",
        "engineer": "internal_stability",
        "finance":  "legal_exposure",
        "ops":      "internal_stability",
        "comms":    "public_exposure",
    }
    return axes.get(role_key, "resolution_score")
```

### 4.4 Voice Assignment Logic

```python
# agents/voice_assignment.py

# 30 HD voices mapped to voice_style categories
# Each agent gets a UNIQUE voice — no duplicates per session

VOICE_STYLE_MAP = {
    "authoritative": ["Orus", "Charon", "Fenrir", "Schedar", "Rasalgethi"],
    "warm":          ["Aoede", "Callirrhoe", "Leda", "Auva", "Erinome"],
    "clipped":       ["Puck", "Achird", "Algenib", "Iapetus", "Gacrux"],
    "measured":      ["Kore", "Umbriel", "Sulafat", "Alnilam", "Sadachbia"],
    "urgent":        ["Zephyr", "Despina", "Enceladus", "Algieba", "Vindemiatrix"],
    "calm":          ["Laomedeia", "Sadaltager", "Pulcherrima", "Zubenelgenubi", "Agenor"],
    "aggressive":    ["Fenrir", "Charon", "Orus", "Rasalgethi", "Schedar"],
}

def assign_voices(agents: list[dict]) -> dict[str, str]:
    """
    Takes the agent list from Scenario Analyst output.
    Returns a dict of { role_key: voice_name }.
    Guarantees no two agents share a voice.
    """
    assigned = {}
    used_voices = set()

    for agent in agents:
        style = agent.get("voice_style", "measured")
        candidates = VOICE_STYLE_MAP.get(style, VOICE_STYLE_MAP["measured"])

        for voice in candidates:
            if voice not in used_voices:
                assigned[agent["role_key"]] = voice
                used_voices.add(voice)
                break
        else:
            # Fallback: pick any unused voice
            all_voices = set(sum(VOICE_STYLE_MAP.values(), []))
            remaining = list(all_voices - used_voices)
            if remaining:
                assigned[agent["role_key"]] = remaining[0]
                used_voices.add(remaining[0])

    return assigned
```

```python
# session_bootstrapper.py
# Called when user submits crisis input. Runs the full startup sequence.

import asyncio
from google.cloud import firestore, pubsub_v1

async def bootstrap_session(crisis_input: str, chairman_id: str) -> str:
    """
    Full session initialization sequence.
    Returns session_id when the room is ready for the chairman to enter.

    SEQUENCE:
    1.  Generate session_id
    2.  Run Scenario Analyst → get full scenario spec
    3.  Write initial crisis_sessions document to Firestore
    4.  Assign unique voices to each agent
    5.  Generate SKILL.md for each agent (parallel)
    6.  Initialize each CrisisAgent (parallel)
    7.  Open Gemini Live WebSocket for each agent (parallel)
    8.  Initialize Observer Agent (reads all public data)
    9.  Schedule World Agent escalation events
    10. Push "session_ready" event → frontend moves to War Room
    """

    session_id = str(uuid.uuid4())[:8].upper()  # e.g. "A3F9B2C1"
    db = firestore.AsyncClient()
    publisher = pubsub_v1.PublisherClient()

    # Step 1: Push "assembling" event to frontend immediately
    await _push_session_event(session_id, "session_status", {
        "status": "assembling",
        "message": "Analyzing crisis scenario..."
    })

    # Step 2: Scenario Analyst
    scenario = await run_scenario_analyst(crisis_input, session_id)

    await _push_session_event(session_id, "session_status", {
        "status": "assembling",
        "message": "Assembling crisis team..."
    })

    # Step 3: Write initial Firestore document
    await db.collection("crisis_sessions").document(session_id).set({
        "session_id": session_id,
        "chairman_id": chairman_id,
        "created_at": firestore.SERVER_TIMESTAMP,
        "status": "assembling",
        "crisis_input": crisis_input,
        "crisis_title": scenario["crisis_title"],
        "crisis_domain": scenario["crisis_domain"],
        "crisis_brief": scenario["crisis_brief"],
        "threat_level": scenario["threat_level_initial"],
        "resolution_score": scenario["resolution_score_initial"],
        "score_history": [scenario["resolution_score_initial"]],
        "posture": {
            "public_exposure": 60,
            "legal_exposure": 45,
            "internal_stability": 50,
            "public_trend": "rising",
            "legal_trend": "stable",
            "internal_trend": "stable",
        },
        "agent_roster": [],
        "agreed_decisions": [],
        "open_conflicts": scenario.get("initial_conflicts", []),
        "critical_intel": scenario.get("initial_intel", []),
        "escalation_events": [],
        "next_escalation_at": None,
        "final_decision": None,
    })

    # Step 4: Voice assignment
    voice_assignments = assign_voices(scenario["agents"])

    # Step 5+6+7: Generate SKILLs and init agents in parallel
    agent_instances = {}

    async def init_one_agent(agent_config: dict):
        role_key = agent_config["role_key"]
        voice = voice_assignments[role_key]
        agent_id = f"{role_key}_{session_id}"

        await _push_session_event(session_id, "agent_assembling", {
            "character_name": agent_config["character_name"],
            "role_title": agent_config["role_title"],
            "identity_color": agent_config["identity_color"],
            "defining_line": agent_config["defining_line"],
            "voice_name": voice,
        })

        # Generate SKILL.md
        skill_md = await generate_skill_md(agent_config, scenario, session_id, voice)

        # Initialize agent
        agent = CrisisAgent(
            session_id=session_id,
            agent_id=agent_id,
            role_config=agent_config,
            skill_md=skill_md,
            assigned_voice=voice,
        )

        # Open Live voice session
        await agent.initialize_live_session()

        # Write to agent_memory
        await agent.memory_ref.set({
            "agent_id": agent_id,
            "session_id": session_id,
            "character_name": agent_config["character_name"],
            "private_facts": [],
            "hidden_agenda": agent_config["hidden_knowledge"],
            "private_commitments": [],
            "previous_statements": [],
            "public_positions": {},
            "contradictions_detected": 0,
            "adk_session_id": agent.adk_session_id,
            "voice_name": voice,
            "voice_session_active": True,
        })

        agent_instances[role_key] = agent
        return agent_config | {"agent_id": agent_id, "voice_name": voice, "status": "idle"}

    # Run all agent inits in parallel
    roster_entries = await asyncio.gather(*[
        init_one_agent(a) for a in scenario["agents"]
    ])

    # Step 8: Observer Agent (special — reads shared state, no voice)
    observer = ObserverAgent(session_id=session_id)
    await observer.start_watching()

    # Step 9: World Agent (timer-based escalation)
    world = WorldAgent(
        session_id=session_id,
        escalation_schedule=scenario["escalation_schedule"]
    )
    await world.start_timer()

    # Update Firestore with full roster
    await db.collection("crisis_sessions").document(session_id).update({
        "agent_roster": list(roster_entries),
        "status": "briefing",
        "next_escalation_at": _compute_next_escalation(scenario["escalation_schedule"])
    })

    # Step 10: Session ready
    await _push_session_event(session_id, "session_ready", {
        "session_id": session_id,
        "crisis_title": scenario["crisis_title"],
        "agent_count": len(roster_entries),
    })

    return session_id
```

```python
# gateway/main.py
# The single entry point. Frontend connects here via WebSocket.

from fastapi import FastAPI, WebSocket, WebSocketDisconnect
from google.cloud import firestore
import asyncio, json

app = FastAPI()

@app.websocket("/ws/{session_id}")
async def war_room_websocket(websocket: WebSocket, session_id: str):
    """
    Frontend connects here once per session.
    Streams ALL events for this session to the frontend.
    This is the ONLY data channel. No REST endpoints for live data.
    """
    await websocket.accept()

    db = firestore.AsyncClient()

    # Firestore real-time listener on session_events subcollection
    events_ref = (
        db.collection("session_events")
          .document(session_id)
          .collection("events")
          .order_by("timestamp")
          .where("consumed_by_frontend", "==", False)
    )

    # Queue for events coming from Firestore listener
    event_queue = asyncio.Queue()

    def on_snapshot(doc_snapshot, changes, read_time):
        for change in changes:
            if change.type.name == "ADDED":
                event_queue.put_nowait(change.document.to_dict())

    # Start the Firestore listener in background
    watch = events_ref.on_snapshot(on_snapshot)

    try:
        # Two concurrent tasks:
        # 1. Forward Firestore events to WebSocket
        # 2. Receive chairman input from WebSocket

        async def forward_events():
            while True:
                event = await event_queue.get()
                await websocket.send_json(event)
                # Mark consumed (optional, for debugging)
                await db.collection("session_events") \
                        .document(session_id) \
                        .collection("events") \
                        .document(event["event_id"]) \
                        .update({"consumed_by_frontend": True})

        async def receive_chairman():
            while True:
                raw = await websocket.receive_text()
                msg = json.loads(raw)

                if msg["type"] == "chairman_audio":
                    # Chairman spoke — route to target agent(s)
                    await handle_chairman_audio(
                        session_id=session_id,
                        audio_b64=msg["audio"],
                        target_agent_id=msg.get("target_agent_id"),
                        transcript=msg.get("transcript", ""),
                    )

                elif msg["type"] == "chairman_command":
                    # FORCE VOTE, DISMISS AGENT, INJECT INTEL, etc.
                    await handle_chairman_command(
                        session_id=session_id,
                        command=msg["command"],
                        params=msg.get("params", {}),
                    )

        await asyncio.gather(forward_events(), receive_chairman())

    except WebSocketDisconnect:
        watch.unsubscribe()
```

```python
# voice/pipeline.py
# How audio flows between chairman, agents, and Live API

"""
VOICE FLOW DIAGRAM:

CHAIRMAN MIC (browser)
       |
       | PCM 16kHz via WebSocket
       |
GATEWAY → routes to target agent's Live session
       |
GEMINI LIVE API (agent's session)
  - VAD detects speech start/end
  - Audio processed natively (no STT step)
  - Agent "thinks" and responds
  - Response streams back as:
    a) Audio chunks (PCM 24kHz) → forwarded to frontend → played in browser
    b) Text transcript (side channel) → pushed as agent_speaking_chunk event
       → appears in Agent Pod transcript
       → consumed by Observer Agent for consistency analysis
       → stored in agent's Firestore memory as previous_statements
       |
       ↓
  INTERRUPTION HANDLING:
  - If chairman speaks while agent is speaking:
    Live API detects barge-in
    Sends interrupted=True signal
    Gateway pushes agent_interrupted event to frontend
    Frontend stops playing audio for that agent
    Chairman audio now goes to gateway
    Gateway routes to relevant agents
"""

async def handle_agent_live_response(
    agent: CrisisAgent,
    session_id: str,
    gateway_websocket_manager,
):
    """
    Runs as a background task for each agent.
    Consumes the agent's Live API response stream.
    Forwards audio + transcript to frontend.
    Feeds transcript to Observer Agent.
    """
    await push_event(session_id, "agent_thinking", {"agent_id": agent.agent_id})

    full_transcript = ""
    audio_chunks = []

    async for response in agent.live_session.receive():

        # ── INTERRUPTED ──────────────────────────────────────────────
        if response.server_content and response.server_content.interrupted:
            await push_event(session_id, "agent_interrupted", {
                "agent_id": agent.agent_id,
                "interrupted_by": "chairman"
            })
            await push_event(session_id, "agent_status_change", {
                "agent_id": agent.agent_id,
                "status": "listening",
                "previous_status": "speaking",
            })
            break

        # ── AUDIO CHUNK ──────────────────────────────────────────────
        if response.data:
            import base64
            audio_b64 = base64.b64encode(response.data).decode()
            audio_chunks.append(response.data)

            # Push audio to frontend for playback
            await push_event(session_id, "agent_speaking_chunk", {
                "agent_id": agent.agent_id,
                "audio_chunk_b64": audio_b64,
                "transcript_chunk": "",
                "is_final": False,
            })

            # First chunk → trigger speaking UI state
            if len(audio_chunks) == 1:
                await push_event(session_id, "agent_speaking_start", {
                    "agent_id": agent.agent_id,
                    "character_name": agent.role_config["character_name"],
                })
                await push_event(session_id, "agent_status_change", {
                    "agent_id": agent.agent_id,
                    "status": "speaking",
                    "previous_status": "thinking",
                })

        # ── TEXT TRANSCRIPT (side channel) ───────────────────────────
        if response.server_content and response.server_content.output_transcription:
            chunk = response.server_content.output_transcription.text
            full_transcript += chunk

            await push_event(session_id, "agent_speaking_chunk", {
                "agent_id": agent.agent_id,
                "transcript_chunk": chunk,
                "audio_chunk_b64": "",
                "is_final": False,
            })

        # ── TURN COMPLETE ─────────────────────────────────────────────
        if response.server_content and response.server_content.turn_complete:
            await push_event(session_id, "agent_speaking_end", {
                "agent_id": agent.agent_id,
                "full_transcript": full_transcript,
            })
            await push_event(session_id, "agent_status_change", {
                "agent_id": agent.agent_id,
                "status": "idle",
                "previous_status": "speaking",
            })

            # Feed to Observer Agent for analysis
            await observer_agent.analyze_statement(
                session_id=session_id,
                agent_id=agent.agent_id,
                transcript=full_transcript,
            )

            # Store in agent's private memory
            await agent.memory_ref.update({
                "previous_statements": firestore.ArrayUnion([{
                    "text": full_transcript,
                    "timestamp": datetime.utcnow().isoformat(),
                }])
            })

            break
```

```python
# agents/observer_agent.py
# Silent agent — no voice, no ADK session.
# Watches all transcripts and Firestore state.
# Outputs insights and trust scores.

from google.adk.agents import LlmAgent
from google.adk.runners import Runner
from google.adk.sessions import InMemorySessionService

OBSERVER_INSTRUCTION = """
You are the Observer Agent in WAR ROOM. You have no voice.
You watch everything. You output structured analysis only.

You receive:
- A recent agent statement (transcript)
- The agent's previous public positions (from their Firestore record)
- The current crisis board state
- The current resolution score

You output a JSON object with this structure:
{
  "trust_delta": number,              // -20 to +10 per statement
  "trust_reason": string,             // one-line explanation
  "insight_type": "contradiction" | "alliance" | "blind_spot" | "mood_shift" | null,
  "insight_title": string | null,
  "insight_body": string | null,
  "agents_referenced": [string],
  "posture_impact": {
    "public_exposure_delta": number,  // -10 to +10
    "legal_exposure_delta": number,
    "internal_stability_delta": number,
  },
  "resolution_score_delta": number,   // -5 to +5
}
"""

class ObserverAgent:
    def __init__(self, session_id: str):
        self.session_id = session_id
        self.db = firestore.AsyncClient()
        self.session_service = InMemorySessionService()

        self.llm = LlmAgent(
            name="ObserverAgent",
            model="gemini-2.5-flash",
            instruction=OBSERVER_INSTRUCTION,
            output_schema=ObserverOutput,  # Pydantic model
        )

        self.runner = Runner(
            agent=self.llm,
            app_name=f"observer_{session_id}",
            session_service=self.session_service,
        )

    async def analyze_statement(
        self,
        session_id: str,
        agent_id: str,
        transcript: str,
    ):
        """
        Runs after every agent turn.
        Produces trust score updates + room intelligence insights.
        """
        # Gather context (READ ONLY)
        memory_doc = await self.db.collection("agent_memory") \
                                  .document(f"{agent_id}_{session_id}") \
                                  .get()
        crisis_doc = await self.db.collection("crisis_sessions") \
                                  .document(session_id).get()

        memory = memory_doc.to_dict()
        crisis = crisis_doc.to_dict()

        # Run Observer LLM
        result = await self.runner.run_async(
            user_id="system",
            session_id=f"observer_{session_id}_{agent_id}",
            new_message=types.Content(
                role="user",
                parts=[types.Part(text=json.dumps({
                    "agent_id": agent_id,
                    "new_statement": transcript,
                    "previous_positions": memory.get("public_positions", {}),
                    "crisis_board": {
                        "agreed": crisis.get("agreed_decisions", []),
                        "conflicts": crisis.get("open_conflicts", []),
                    },
                    "current_resolution_score": crisis.get("resolution_score", 50),
                }))]
            )
        )

        output = json.loads(result.final_response.text)

        # Push trust score update
        if output.get("trust_delta") is not None:
            current_doc = await self.db.collection("crisis_sessions") \
                                       .document(session_id).get()
            roster = current_doc.to_dict().get("agent_roster", [])

            for agent in roster:
                if agent["agent_id"] == agent_id:
                    new_score = max(0, min(100,
                        agent.get("trust_score", 70) + output["trust_delta"]
                    ))
                    await push_event(session_id, "trust_score_update", {
                        "agent_id": agent_id,
                        "score": new_score,
                        "delta": output["trust_delta"],
                        "reason": output["trust_reason"],
                    })

        # Push insight if generated
        if output.get("insight_type"):
            await push_event(session_id, "observer_insight", {
                "insight_id": str(uuid.uuid4()),
                "type": output["insight_type"],
                "title": output["insight_title"],
                "body": output["insight_body"],
                "agents_referenced": output["agents_referenced"],
            })

        # Update posture
        posture_impact = output.get("posture_impact", {})
        if any(v != 0 for v in posture_impact.values()):
            await _update_posture(session_id, posture_impact)

        # Update resolution score
        if output.get("resolution_score_delta", 0) != 0:
            await _update_resolution_score(
                session_id,
                output["resolution_score_delta"],
                "Agent statement impact"
            )
```

```python
# agents/world_agent.py
# Timer-based. Fires escalation events at scheduled intervals.
# Also generates reactive feed items in response to agent speech.

class WorldAgent:
    def __init__(self, session_id: str, escalation_schedule: list[dict]):
        self.session_id = session_id
        self.escalation_schedule = escalation_schedule
        self.db = firestore.AsyncClient()

        self.llm = LlmAgent(
            name="WorldAgent",
            model="gemini-2.5-flash",
            instruction=WORLD_AGENT_INSTRUCTION,
        )

    async def start_timer(self):
        """Schedule escalation events as asyncio tasks."""
        for event in self.escalation_schedule:
            delay = event["delay_minutes"] * 60
            asyncio.create_task(
                self._fire_escalation(delay, event)
            )

    async def _fire_escalation(self, delay_seconds: float, event: dict):
        await asyncio.sleep(delay_seconds)

        event_id = str(uuid.uuid4())

        # Write to crisis board intel
        await self.db.collection("crisis_sessions") \
                     .document(self.session_id) \
                     .update({
                         "critical_intel": firestore.ArrayUnion([{
                             "intel_id": event_id,
                             "text": event["event_text"],
                             "source": event["type"].upper(),
                             "timestamp": datetime.utcnow().isoformat(),
                             "is_escalation": True,
                         }])
                     })

        # Push escalation event (triggers full-screen flash on frontend)
        await push_event(self.session_id, "crisis_escalation", {
            "event_id": event_id,
            "text": event["event_text"],
            "type": event["type"],
            "next_escalation_in_seconds": 0,
        })

        # Push as feed item
        await push_event(self.session_id, "feed_item", {
            "feed_id": str(uuid.uuid4()),
            "text": event["event_text"],
            "source_name": "📡 WORLD FEED",
            "source_type": event["type"].upper(),
            "metric": f"⚡ CRISIS ESCALATION · just now",
            "is_hot": True,
        })

        # Update resolution score negatively
        await _update_resolution_score(
            self.session_id,
            -8,
            f"Escalation: {event['event_text'][:40]}..."
        )

        # Update threat level if score crosses threshold
        await _check_threat_level(self.session_id)

        # Notify all agents via Pub/Sub so they react
        await _broadcast_to_agents(self.session_id, {
            "type": "world_escalation",
            "event_text": event["event_text"],
            "event_type": event["type"],
        })
```

## 13. MEMORY LEAKAGE PREVENTION — SUMMARY

```
✅ Each agent has its own InMemorySessionService instance
   → ADK session data never mingles between agents

✅ Each agent has its own Firestore collection prefix
   → /agent_memory/{agent_id}_{session_id}
   → Firestore security rules: agent service account can only
     read/write documents matching its own agent_id prefix

✅ Agents can only READ one field from each other:
   read_other_agent_last_statement() → returns only the last
   public statement string, NOT the full memory document

✅ Hidden knowledge (hidden_agenda field) is stored ONLY in
   /agent_memory — never written to /crisis_sessions

✅ ADK session state uses no user: or app: prefix scoping
   → All state is session-scoped (wiped when session ends)

✅ Each agent's Gemini Live session is a separate WebSocket
   → No shared context window between agents' voice sessions

✅ Observer Agent reads ONLY transcripts and public crisis state
   → Cannot access any agent's private memory collection

✅ World Agent has NO memory of agent statements
   → Only reads the crisis scenario from /crisis_sessions
```
